# ProcessDyes.m Usage Guide

This script extracts spectral information from RainbowSTORM raw microscopy data and localizations. It produces a `.mat` file containing raw and background-subtracted spectra for each detected localization.

## Prerequisites

Before running `ProcessDyes.m`, you need to prepare the following files:

### 1. Raw Microscopy Data (.nd2 format)
- Spectral microscopy data with spatial and spectral dimensions
- Format: (x, y, frame) array
- Should contain both 0th order (spatial) and 1st order (spectral) information

### 2. Localization Data (CSV format)
- Generated by running **ThunderSTORM** on the raw `.nd2` file
- File must be named exactly the same as the `.nd2` file with `.csv` extension
- **Important**: When running ThunderSTORM, select the ROI that covers the **0th order region only** (spatial region, not spectral)
- Must have at minimum these columns:
  - `frame` - frame number
  - `x [nm]` - x-coordinate in nanometers
  - `y [nm]` - y-coordinate in nanometers
  - `intensity [photon]` - intensity in photons

### 3. Spectral Calibration File (speccali.mat)
- Contains wavelength-to-pixel mapping information
- Must be placed in the same directory as your raw data
- Should contain a variable named `speccali` with a field `fy` (pixel coordinates of calibration points)
- Generate this using the spectral calibration script on a calibration sample with known emission wavelengths

## Setup Instructions

### Step 1: Prepare Your Data Directory
Create a folder with the following structure:
```
/your/data/path/YYYY-MM-DD/
├── FOV001.nd2                  (raw spectral data)
├── FOV001.csv                  (ThunderSTORM localizations)
├── FOV002.nd2
├── FOV002.csv
└── speccali.mat                (spectral calibration)
```

### Step 2: Configure the Script Parameters
Open `ProcessDyes.m` and modify the section labeled `%% CHANGE THIS PART %%`:

```matlab
pathdate = '2026-02-05';        % Date folder name
pathname = '/Users/weihong/Desktop/temp/data/2026-02-05/';  % Full path to data directory

px_size = 110;                  % Pixel size in nanometers
order0_roi = [0, 975, 352, 377];  % [x, y, width, height] of 0th order region, in pixels

debug = true;                   % Set to true to see visualization plots

wl_min = 535;                   % Minimum wavelength range (nm)
wl_max = 800;                   % Maximum wavelength range (nm)
roi_radius = 2;                 % Radius around localization center (pixels, not including center pixel)
```

#### Parameter Details:
- **pathdate**: Corresponding to your data directory folder name
- **pathname**: Full path where your `.nd2`, `.csv`, and `speccali.mat` files are located
- **px_size**: Typically 110 nm for your setup (from your microscopy specifications)
- **order0_roi**: The rectangular region you selected in ThunderSTORM for the 0th order
  - `x`: leftmost x-coordinate
  - `y`: topmost y-coordinate
  - `width`: width in pixels
  - `height`: height in pixels
- **wl_min/wl_max**: Expected wavelength range for your spectral region
- **roi_radius**: How many pixels around each localization to include in the spectrum

### Step 3: Run the Script
```matlab
ProcessDyes  % or F5 in MATLAB/Octave
```

### Step 4: Check the Output
The script generates:
1. **Debug Figures** (if `debug = true`):
   - Figure 1: Overview of 0th and 1st order regions with localizations
   - Figure 2: Detailed view of top 10 brightest localizations with extracted spectra

2. **Output File**:
   - Location: `processed_files/FOV001.mat` (automatically created)
   - Contains: `output_data` structure with spectra and metadata for all localizations

## Output Format

The saved `.mat` file contains a structure `output_data` with the following fields:

```matlab
output_data.frame              % Frame number for each localization
output_data.x_px              % X-coordinate in pixels
output_data.y_px              % Y-coordinate in pixels
output_data.x_nm              % X-coordinate in nanometers
output_data.y_nm              % Y-coordinate in nanometers
output_data.intensity          % Intensity in photons
output_data.spectrum_raw       % Raw spectrum (n_locs × n_wavelengths)
output_data.spectrum_bgsub     % Background-subtracted spectrum (n_locs × n_wavelengths)
output_data.wavelengths        % Wavelength values for each pixel (n_locs × n_wavelengths)
```

## Loading and Using the Results

```matlab
% Load the processed data
load('processed_files/FOV001.mat');

% Access a specific localization
loc_idx = 1;
spectrum = output_data.spectrum_bgsub(loc_idx, :);
wavelengths = output_data.wavelengths(loc_idx, :);

% Plot the spectrum
plot(wavelengths, spectrum);
xlabel('Wavelength (nm)');
ylabel('Intensity (photons)');
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "File does not exist" error | Check that `.nd2` and `.csv` files have matching names and are in the correct folder |
| "speccali.mat not found" | Ensure `speccali.mat` is in the same directory as your data files |
| Empty or all-zero spectra | Check that localizations are within the spectral region bounds; adjust `wl_min/wl_max` if needed |
| "Index out of bounds" warning | Some localizations may be near the edges; they are automatically skipped |
| CSV columns not found | Verify ThunderSTORM CSV has columns: `frame`, `x [nm]`, `y [nm]`, `intensity [photon]` |

## Notes

- The script processes **all frames and all localizations** in the CSV file
- Both raw and background-subtracted spectra are extracted for comparison
- Wavelength calibration is performed using the `speccali.mat` file and the DWP (Dispersive Wave Plate) functions
- Processing may take several minutes for large datasets
